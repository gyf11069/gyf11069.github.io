远程推送代码时出现的问题

# 使用 Git 推送代码时出现的问题

# 1、问题

$ git push
Logon failed, use ctrl+c to cancel basic credential prompt.
remote: Support for password authentication was removed on August 13, 2021. Please use a personal access token instead.
remote: Please see https://github.blog/2020-12-15-token-authentication-requirements-for-git-operations/ for more information.
fatal: Authentication failed for 'https://github.com/gyf11069/gyf11069.github.io.git/'
1
2
3
4
5

大致的意思是从2021.8.13开始就不支持通过账号密码的方式远程连接Github，需要用个人令牌进行远程连接。

注：个人访问令牌 (opens new window)只能用于 HTTPS Git操作。如果您的存储库使用 SSH远程 URL，则需要将远程从 SSH 切换到 HTTPS (opens new window)。

于是我就到上述给出的地址https://github.blog/2020-12-15-token-authentication-requirements-for-git-operations/查看是否有解决办法，里面主要说明了一下 Git 操作的令牌认证要求。我找到“你需要做什么标题处”发现可以使用 HTTPS（推荐）或 SSH 密钥个人令牌进行验证。我就想可能是我配置的 SSH 密钥出问题了，我就测试我的 SSH 连接 (opens new window)。发现果然有问题。（生成的 SSH 密钥一年未使用的话会被自动删除。注意先查看一下是否过期）

HP@GYUNF MINGW64 /e/blog/gyf11069.github.io (main)
$ ssh-add -l -E sha256
The agent has no identities.
1
2
3

查询是否添加了ssh-agent时报错了，于是我就接着按照测试我的 SSH 连接 (opens new window)的步骤进行操作，使用下面的代码解决了问题。

HP@GYUNF MINGW64 /e/blog/gyf11069.github.io (main)
$ ssh-add ~/.ssh/id_rsa
Identity added: /c/Users/HP/.ssh/id_rsa (1224254097@qq.com)
1
2
3

当我输入如下测试 SSH连接时又报错了：

$ ssh -T git@github.com
1

错误信息如下：（这个信息应该不算是报错信息）

Hi gyf11069! You've successfully authenticated, but GitHub does not provide shell access.

您已成功通过身份验证，但 GitHub 不提供shell访问权限。

我将报错信息复制到百度，查看是否有解决方案，找了一个比较靠谱的方案解决了问题。但是没有解释具体是什么原因造成的，所以我继续在 Github文档中寻找答案，最终在管理远程存储库 (opens new window)找到的原因。

原因分析：出现上述问题是因为我设置的远程存储库的 URL 没有一一对应导致的。我克隆仓库时是使用 HTTPS 的方式克隆的，我提交是使用 SSH URL 的方式进行的。所以产生了矛盾。

# 2、解决方案

git remote set-url origin git@github.com:gyf11069/gyf11069.github.io.git

然后再输入git push就行了。

该git remote set-url命令作用是更改现有的远程存储库 URL。（提示：有关 HTTPS 和 SSH URL 之间区别的信息，请参阅“关于远程存储库 (opens new window)”。）

该git remote set-url命令有两个参数：

  • 现有的远程名称。例如，origin or upstream是两个常见的选择。

  • 远程的新 URL。例如：

    • 如果您要更新以使用 HTTPS，您的 URL可能如下所示：

      https://github.com/用户名/仓库名.git
      1

    • 如果您要更新以使用 SSH，您的 URL可能如下所示：

      git@github.com:用户名/仓库名.git
      1

我使用的克隆和推送的命令：

# 克隆
git clone https://github.com/用户名/仓库名.git
#推送,先将远程 URL 修改为 SSH URL
git remote set-url origin git@github.com:用户名/仓库名.git
# 推送至远程仓库
git push

# 3、总结

在Github使用git克隆项目时使用哪种方式克隆，就使用哪种方式推送源代码到Github仓库。
